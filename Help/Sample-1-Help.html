XML Sample-1

 

This samples shows the basics of XML parsing. The sample does the following:

 

  1. Creates an XML Object
  2. Assigns a default namespace to it
  3. Assigns the document to an existing XML document and loads it
  4. Reads through the current document looking for Customers and
  5. For each customer find the customer name, phone and state
  6. For each customer state, convert it from a state name to a state ID and place it back in the XML document
  7. Save the altered XML document under a new name.

 

This sample is full of comments…read them.

 

As with all the samples we are using an embedded IE5 control to display the XML files. Don’t get confused and think this is what you are supposed to do with XML files. The IE5 browser just happens to be capable of displaying XML files in a rudimentary tree view format. You are also encouraged to view these XML documents using other XML editors. You might want to use Microsoft’s XML Notepad – a simple XML editor that can be downloaded for free from various sources.

 

In this example you can compare the original XML file (data\Customer.XML) with the XML file created by this application (data\NewCustomer.Xml). The two files will be identical except the Customer.xml shows state names (e.g. California) and the NewCustomer.xml has been altered to show state codes (CA). Viewing these two just verifies that the application works.

 

The customer XML document is not necessarily designed to represent the DataFlex customer data. In fact, this document has its origins outside of DataFlex. In this sample we never even open a data base file. If we wanted we could use the XML file to make changes in an existing data file or to use data from our files to update the XML document. These kinds of updates are managed by the code that your write.

 

Study and understand this sample before moving on to the others. It has all of the basics of XML manipulation. You:

  1. Create and destroy nodes.
  2. You create a Nodelist using a simple search pattern.
  3. You alter XML nodes and place them back into the document.
  4. You load and save XML documents.
