XML Sample-2

 

This samples writes data from a DDO to an XML file. In this example we create an XML document named MyCustomer.xml that contains some information from our Customer data file.  For each customer record we want to create a customer node that looks like this:

 

Customer (element)

Customer-Id = id (attribute)

Name = name(element)

Address (element)

Street = street (element)

City = city (element)

State = state (element)

Zip = zip (element)

Telephone = phone (element)

 

 

We have purposefully not created a one-to-one relationship between the data in the file and the data in the XML file nor is the structure of the XML file the same as the file. When you create an XML file, the structure and contents you use should not be based on the structure of your data. It should be based on the needs of the processes that are exchanging data.

 

In this sample we have an option that allows you to enter duplicate customer entries or to skip duplicates. This is done by using XML’s Document Object Model pattern search capability.

 

Again we use the IE5 browser to let you see the results of the sample. You can choose to first delete any existing XML or work with the existing file.
